🔑 Key Management

The server uses a Fernet (AES-128-CBC) symmetric key stored on the server filesystem.

  • ✅ Each encryption/decryption uses the same server-side key.
  • ✅ The key persists across restarts when mounted as a volume.
  • ⚠️ Generating a new key will make all previously encrypted data unreadable.

⚠️ Danger Zone

Generating a new key is irreversible. Only do this if you want to start fresh.

🔐 Your data is encrypted locally. No data is stored on our servers.
